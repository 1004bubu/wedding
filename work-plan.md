## Phase별 작업 계획

### Phase 0 · 선행 작업 — 번들 해체 및 재구현 방향 결정 (난이도: 중)

현재 파일은 Claude Design 번들 포맷(12MB)입니다. Firebase·이미지·커스텀 기능을 추가하려면 직접 수정이 불가하므로, 디자인을 참고해 새 HTML/CSS/JS로 재구현을 권장합니다.

- **[필수]** 브라우저에서 `invitation.html` 열어 전체 UI 캡처 (커버, 갤러리, 인사말, 일시장소, 방명록, RSVP, 계좌 섹션 구조 파악)
  - 디자인 토큰(색상, 서체, 간격) 추출 → 새 파일에 적용
- **[필수]** 새 파일 `index.html` 생성, 모듈 구조 결정 (단일 파일 vs JS 분리)
  - Firebase SDK 로드, 이미지 자산 구조 설계

---

### Phase 1 · 이미지 — 메인 이미지 · 갤러리 구현 (난이도: 하)

커버 사진 및 갤러리 사진을 추가합니다. 이미지는 Firebase Storage에 업로드하거나 base64로 인라인 삽입합니다.

- **[필수]** 메인(커버) 이미지 삽입 — 전체 화면 또는 카드형
  - 권장 비율: 3:4 세로형 (모바일 최적화)
- **[필수]** 갤러리 구현 — 스와이프 슬라이더 또는 그리드 레이아웃
  - 터치 스와이프 지원 (CSS scroll-snap 사용)
- **[권장]** 이미지 최적화 — WebP 변환, Lazy Load 적용
  - 모바일 환경에서 로딩 속도 확보

---

### Phase 2 · Firebase — 방명록 · 참석 여부 연동 (난이도: 중상)

Firestore로 방명록 및 RSVP 데이터를 저장합니다. 참석 집계는 Firebase Console에서 직접 확인하며, 청첩장에는 노출하지 않습니다.

- **[필수]** Firebase 프로젝트 생성 및 Firestore 활성화
  - Firebase Console → 새 프로젝트 → Firestore Database(프로덕션 모드)
- **[필수]** Firestore 데이터 구조 설계

**Firestore 컬렉션 구조**

`/guestbook/{docId}`

| 필드 | 타입 |
|------|------|
| name | string |
| message | string |
| side | "groom" \| "bride" |
| createdAt | timestamp |

`/rsvp/{docId}`

| 필드 | 타입 |
|------|------|
| name | string |
| attending | boolean |
| guestCount | number |
| meal | "yes" \| "no" |
| phone | string (선택) |
| createdAt | timestamp |

- **[필수]** 보안 규칙 설정 — 쓰기 허용, 수정·삭제는 인증 사용자만
  - 방명록은 전체 공개 읽기 / RSVP는 쓰기만 허용 (읽기는 관리자 전용)
- **[참고]** 참석 집계 확인 방법 — Firebase Console에서 필터링
  - 청첩장 내 노출 없음. `attending=true` 필터 후 `guestCount` 합산으로 확인

### Phase 3 · 연락처 — 전화/문자 버튼 → 카카오톡 연동 (난이도: 하)

기존 전화·문자 버튼을 카카오톡 채팅 버튼으로 교체합니다. 방식에 따라 별도 설정이 필요합니다.

| 방식 | 장점 | 단점 | 준비물 |
|------|------|------|--------|
| **오픈채팅 링크** ⭐추천 | 설정 즉시 가능, 별도 인증 불필요 | 익명성 있음 | 오픈채팅방 URL |
| 카카오채널 | 1:1 채팅 지원, 구분 관리 용이 | 채널 개설 필요, 승인 시간 소요 | 카카오 비즈니스 계정 |
| 개인 카카오링크 | 가장 자연스러움 | 개인 ID 직접 노출 | 카카오 ID 확인 |

- **[필수]** 신랑측 / 신부측 각각 카카오 버튼 구현
  - 기존 전화·문자 버튼 2개를 카카오 버튼으로 교체

---

### Phase 4 · 실제 정보 — 계좌번호 업데이트 · 약도 결정 (난이도: 하)

예시 계좌를 실제 정보로 교체하고, 약도 표시 방식을 확정합니다.

- **[정보 필요]** 계좌 정보 입력 — 아래 항목 확인 후 반영

**계좌 입력 템플릿**

| 구분 | 입력 항목 |
|------|-----------|
| 신랑측 | 은행명 · 계좌번호 · 예금주 / 부모님 계좌 포함 여부 |
| 신부측 | 은행명 · 계좌번호 · 예금주 / 부모님 계좌 포함 여부 |

**약도 옵션 비교**

| 방식 | 장점 | 단점 |
|------|------|------|
| 지도 이미지 | 오프라인에서도 표시, 디자인 통일성 | 확대·이동 불가, 이미지 직접 제작 필요 |
| **카카오지도 임베드** ⭐추천 | 실시간 길 찾기 연동, 구현 간단(iframe) | 외부 도메인 의존 |
| 네이버지도 링크 | 국내 사용자 친숙, 대중교통 정보 풍부 | 청첩장 이탈 발생 |

---

### Phase 5 · 마무리 — 테스트 · 배포 (난이도: 하)

모바일 실기기 검수 후 Firebase Hosting 또는 GitHub Pages로 배포합니다.

- **[필수]** iOS / Android 실기기 테스트 — 카카오, 방명록, RSVP, 갤러리 스와이프 동작 확인
- **[필수]** 카카오톡 공유 미리보기 설정 (OG 태그: 제목, 설명, 썸네일 이미지)
- **[권장]** Firebase Hosting 배포 — 커스텀 도메인 연결(선택)
  - 또는 GitHub Pages (Firebase 없이 정적 호스팅)
- **[권장]** Firebase Console 접근 권한 설정 — 두 분 모두 관리자 계정 등록
